- Purpose: Product Catalog sample that proves out the Blazor Enterprise Architecture / Vertical Slice Architecture guide by implementing CQRS + DDD slices for typical product CRUD and bulk operations.
- Tech stack: C#/.NET 8-9 (solution + host), Blazor Server UI, MediatR, FluentValidation, EF Core (SQL Server & PostgreSQL targets), Dapper, SignalR, Serilog, Prometheus.
- Structure: `src/ProductCatalog` holds bounded-context code with `Features/<Feature>/{Application,UI}` slices and BC-level shared code under `src/ProductCatalog/Shared`. Cross-BC libraries live under `src/Shared/{Kernel,Application,Domain,Infrastructure}`. `src/ProductCatalog.Host` is the only executable (DI, middleware, SignalR, layout). Tests live in `tests/ProductCatalog.*Tests` split by layer plus Playwright E2E. Extensive architecture docs live under `docs/` and AI pattern catalogs under `catalog/`.
- Data & runtime dependencies: PostgreSQL via `compose.yml` (podman) powers persistence; EF Core migrations live in `src/ProductCatalog/Shared/Infrastructure/Persistence`. Host applies migrations on first run and seeds default accounts.
